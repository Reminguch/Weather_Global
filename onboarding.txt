Weather_global Cluster Onboarding (Della)

Project root:
/scratch/gpfs/DABANIN/iv9432/Weather_global

1) One-time setup

From the cluster login node (e.g. della.princeton.edu)

cd /scratch/gpfs/DABANIN/iv9432/Weather_global
source scripts/graphcast_env.sh --setup

This creates conda env `graphcast311` (Python 3.11), installs:
- jax[cuda12]
- graphcast (from GitHub)
- netCDF4, xarray (for NetCDF data exploration, e.g. scripts/explore_graphcast_dataset.ipynb)

2) Use environment in each new shell

cd /scratch/gpfs/DABANIN/iv9432/Weather_global
source scripts/graphcast_env.sh

2b) Show graphcast311 in Jupyter/notebook kernel list (one-time, with env activated)

pip install ipykernel
python -m ipykernel install --user --name graphcast311 --display-name "Python (graphcast311)"
Then in Cursor/VS Code: notebook kernel picker → "Python (graphcast311)" or "Select Another Kernel" → Jupyter Kernel → graphcast311.

3) Submit GPU verification job

cd /scratch/gpfs/DABANIN/iv9432/Weather_global
sbatch scripts/verify_gpu_jax.slurm

4) GPU compute node allocation:

4.0 Check available GPU nodes:
example: `sinfo -o "%P %G %f"`

4.1 then use salloc to get a GPU node.
example: salloc --nodes=1 --ntasks=1 --mem=32G --time=02:00:00 --gres=gpu:a100:1


4.2 On the compute node: env + start Jupyter

cd /scratch/gpfs/DABANIN/iv9432/Weather_global

source scripts/graphcast_env.sh
jupyter notebook --no-browser --port=8888 --ip=0.0.0.0

Copy the URL it prints, e.g.
http://della-l04g4:8888/?token=e2951779957f4999e647ff17d04044225d01096678f1b500
(you need the token).
4.3 From your local machine: SSH port forward
The compute hostname (e.g. della-l04g4) is only reachable from the cluster. So on your local machine (where Cursor runs) open a tunnel:
ssh -L 8889:della-i14g19:8888 iv9432@della.princeton.edu
Use the actual compute node name from step 1 in place of della-l04g4. Leave this SSH session open.

5) Check logs

- logs/verify-gpu-jax-<jobid>.out
- logs/verify-gpu-jax-<jobid>.err

Expected in `.out`:
- `nvidia-smi` lists GPU
- `Using conda env: graphcast311`
- `jax_backend: gpu`
- `jax_devices` contains at least one `CudaDevice(...)`

Reference successful output on this cluster:
- node example: `della-l05g4`
- gpu example: `NVIDIA A100 80GB PCIe`
- jax example: `jax_backend: gpu`, `jax_devices: [CudaDevice(id=0)]`

5) If JAX shows CPU only

- Confirm the job actually ran on a GPU node.
- Confirm `nvidia-smi` succeeds inside that job.
- Re-run `source scripts/graphcast_env.sh` and submit again.

6) For Python jobs in Slurm

Use the same environment script at the start of your job script:

cd /scratch/gpfs/DABANIN/iv9432/Weather_global
source scripts/graphcast_env.sh
python your_script.py

7) Stage WB2 data locally (for compute nodes without direct GCS access)

Run this on a login/proxy-enabled context, then train from local Zarr on compute nodes.

Bootstrap local WB2 dataset (1.0deg, 13 levels):

cd /scratch/gpfs/DABANIN/iv9432/Weather_global
source scripts/graphcast_env.sh
python scripts/stage_wb2_era5_yearly_append.py

Append one next missing year later:

python scripts/stage_wb2_era5_yearly_append.py

Dry-run (no writes):

python scripts/stage_wb2_era5_yearly_append.py --dry-run

8) Download local last-30-day dataset (optional)

Useful for short recency-focused evaluation datasets.

python scripts/download_wb2_last30d.py

Default output:
- data/graphcast/graphcast/dataset/wb2_res1_levels13_last30d.zarr

9) Train/fine-tune from local datasets

Fine-tune on selected years, validate on trailing 30 days:

python -u scripts/training/finetune_graphcast.py \
  --train-path data/graphcast/graphcast/dataset/wb2_res1_levels13_1979_2021.zarr \
  --train-start-year 1979 \
  --train-end-year 2021 \
  --val-days 30

Res2 training split:
- train: all years in local dataset except 2021
- val: year 2021

python -u scripts/training/train_graphcast_res2_stream.py \
  --data-path data/graphcast/graphcast/dataset/wb2_res1_levels13_1979_2021.zarr \
  --val-year 2021
